Anagram check-1

Problem Statement:

Given two strings, determine if they are anagrams of each other.

Illustration:

Consider the following examples:

Example 1:

String 1: "listen"
String 2: "silent"

Explanation: Both strings contain the same characters with the same frequency, so they are anagrams.

Example 2:

String 1: "hello"
String 2: "world"

Explanation: The characters and their frequencies in the two strings differ, so they are not anagrams.

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Strings
  • Loops
  • Conditional statements

{
    int i, j, check=1, find;

    for(i=0; str1[i]!='\0'; ++i)
    {
        find = 0;
        for(j=0; str2[j]!='\0'; ++j)
        {
            if(str1[i]==str2[j])
            {
                str2[j]='*';

Python Solution :


def anagram(str1, str2):
    str2 = list(str2)
    for i in range(len(str1)):
        find = False
        for j in range(len(str2)):
            if str1[i] == str2[j]:
                str2[j] = '*'
                find = True
                break
        if not find:
            print("\nNot Anagrams!\n")
            return
    print("\nBoth are Anagrams!\n")

def main():
    str1 = input("Enter string 1: ")
    str2 = input("Enter string 2: ")

    if len(str1) == len(str2):
        anagram(str1, str2)
    else:
        print("\nLengths are not equal\n")

if __name__ == "__main__":
    main()

Explanation:

1. Take two strings as input from the user.
2. Iterate through each character of the first string.
3. For each character, search for a matching character in the second string.
4. If a match is found, mark the character in the second string and continue.
5. If all characters are matched, the strings are anagrams; otherwise, they are not.
6. If a character in the first string is not found in the second string, they cannot be anagrams.
7. Print the result based on the outcome of the search.

Time and Space Complexity:

The time complexity of the solution is O(n^2), where n is the length of the strings. This is because of the nested loops used to compare each character of both strings.
