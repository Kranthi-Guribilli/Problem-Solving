Lowercase to uppercase

Problem Statement:

Given a string of characters, convert the string into its uppercase.

What to do??

Input: "hello"

Output: "HELLO"

Input: "kushal Is My ClassMate."

Output: "KUSHAL IS MY CLASSMATE."

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Basic arithmetic operations.
  • ASCII Table (Character Manipulation).
  • loops and conditional statements.

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX 80

void conversion(char *line);

int main(){
    char line[MAX];

    printf("Enter a string : ");
    gets(line);

    conversion(line);

    printf("%s", line);

    return 0;
}

void conversion(char *line){
    int i = 0;
    while (line[i]){
        if (line[i]>='a' && line[i]<='z'){
            line[i] = line[i]-32;
            i++;
        }else{
            i++;
            continue;
        }
    }
}

Python Solution :


def conversion(line):
    result = ""
    for char in line:
        if 'a' <= char <= 'z':
            result += chr(ord(char) - 32)
        else:
            result += char
    return result

def main():
    line = input("Enter a string: ")
    converted_line = conversion(line)
    print(converted_line)

if __name__ == "__main__":
    main()

Explanation:

1. The program defines a function `conversion` that converts lowercase letters in a string to uppercase.
2. It takes a string `line` as input and iterates through each character.
3. If the character is a lowercase letter (ASCII value between 'a' and 'z'), it subtracts 32 from its ASCII value to convert it to uppercase.
4. The conversion is done in-place, directly modifying the input string.

Time and Space Complexity:

Time Complexity:It is O(n), where n is the length of the input string. This is because the code iterates through each character of the string once.

Space Complexity:O(1) because the code only uses a constant amount of extra space regardless of the input size. There are no additional data structures or dynamic memory allocations involved.

Why not just use the built-in methods?:

While it's advisable to utilize built-in string methods for efficiency, there's immense value in learning the hard way. This approach helps build intuition and fosters natural problem-solving skills. By exploring brute-force solutions, you gain a deeper understanding of the necessity for advanced solving techniques and data structures such as Dynamic Programming, Bit Manipulation, priority queues, graph algorithms, and more.
