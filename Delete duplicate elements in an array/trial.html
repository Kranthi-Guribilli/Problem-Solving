Delete Duplicate Elements In An Array

Problem Statement:

Read an array of integers from the user and Delete all the duplicate elements from it.

What to do??

Input:
Enter your array size : 10
Enter array elements : 1 2 3 4 2 6 2 8 9 3

Output: 1 2 3 4 6 8 9

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Arrays, shifting elements
  • Looping

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>

int main(){
    int arr[100];
    int i, j, k, l, size, pos;

    printf("Enter number of elements  : ");
    scanf("%d", &size);
    printf("Enter array elements : ");

    for (i=0; i<size; ++i){
        scanf("%d", &arr[i]);
    }

    for (i=0; i<size; ++i){
        for (j=i+1; j<size; ++j){
            if (arr[i] == arr[j])
            {
                for (k=j; k<size; ++k){
                    arr[k] = arr[k+1];
                }
                size--;
                j--;
            }
        }
    }

    for (i=0; i<size; ++i){
        printf("%d ", arr[i]);
    }

    return 0;
}

Explanation:

1) The program iterates through each element of the array.

2) For each element at index i, it compares with all the elements at indices greater than i to check for duplicates.

3) If a duplicate is found at index j, it shifts all elements from index j+1 to the end of the array one position to the left to remove the duplicate.

4) The variable size is decremented to reflect the removal of the duplicate element.

5) The inner loop index j is decremented to recheck the current index i with the next element after shifting.

This approach ensures that only unique elements remain in the array after processing. This solution effectively removes duplicates from the array by shifting elements and adjusting the size of the array accordingly. However, this approach may not be the most efficient for large arrays with many duplicates, as it involves nested loops and shifting elements.

Time and Space Complexity:

Time Complexity:O(n^2), where n is the number of elements in the array.

Space Complexity:O(n), where n is the number of elements in the array. ofcourse we declared the maxsize as 100. so you may say the space complexity is O(1) but generally speaking, it's O(n)
