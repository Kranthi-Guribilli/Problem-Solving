left-aligned Odd Number Pyramid pattern

Problem Statement:

Write a C program that generates a left-aligned Odd Number Pyramid with'*' symbols.

Illustration:

Enter no of rows : 4

Output Pattern:

*
* * *
* * * * *
* * * * * * *

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Basics of loops, Nested loops

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>

/*

*
* * *
* * * * *
* * * * * * *

*/

int main()
{
    int row, i, col, j;

    printf("Enter number of rows : ");
    scanf("%d", &row);

    for (i = 1; i <= row; i++){
        for (j = 1; j <= (2*i)-1; j++){
            printf("@ ");

        }
        printf("\n");

    }
    return 0;
}

Python Solution :


row = int(input("Enter number of rows: "))
for i in range(1, row + 1):
    for j in range(1, (2 * i) - 1 + 1):
        print("* ", end="")
    print()

Explanation:

1. Prompt the user to enter the number of rows for the pattern.

2. Use nested loops to iterate over each row and column of the pattern.

3. In the outer loop (for i), iterate from 1 to the number of rows.

4. In the inner loop (for j), iterate from 1 to (2*i) - 1 to generate the increasing odd numbers of asterisks.

5. Print an asterisk (*) for each iteration of the inner loop.

6. Print a newline character at the end of each row to move to the next row.

j = 0 j = 1 j = 2 j = 3 j = 4 j = 5 j = 6
i = 0 *
i = 1 * * *
i = 2 * * * * *
i = 3 * * * * * * *

Time and Space Complexity:

Time Complexity: O(n^2) - where "n" is the number of rows. This is because there are nested loops iterating over each element of the pattern.

Space Complexity: O(1) - There is no extra space used that depends on the input size.

Challenge:

An additional challenge could be to modify the program to allow printing the same pattern but flipped horizontally.
