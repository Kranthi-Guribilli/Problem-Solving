Decimal to Binary

Problem Statement:

Given a decimal number. Output it's Binary equivalent.

What to do??

Input:
Enter your Decimal Number : 13

Output: 1101
13 is written as 1101 in binary.

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Basic arithmetic operations.
  • Number conversions.

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>

long long int bin(int deci){
    int p=1, sum=0;

    while(deci){
        sum = sum + (deci%2)*p;
        deci = deci/2;
        p=p*10;
    }
    return sum;
}

int main(){
    int deci;

    printf("Enter your Decimal Number : ");
    scanf("%d", &deci);

    printf("The Binary Equivalent of %d is : %lld", deci, bin(deci));

    return 0;
}

Python Solution :


def bin(deci):
    p = 1
    sum = 0
    while deci:
        sum = sum + (deci % 2) * p
        deci = deci // 2
        p = p * 10
    return sum

deci = int(input("Enter your Decimal Number : "))
print("The Binary Equivalent of", deci, "is :", bin(deci))

Explanation:

1. To convert a decimal number to binary, we start by initializing two variables: `p` and `sum`.

2. `p` represents the position of the current binary digit we're evaluating, starting from the rightmost (least significant) digit. We initialize it to 1, as it corresponds to the 2^0 position.

3. `sum` will hold the binary representation of the decimal number.

4. We enter a loop that continues until the decimal number becomes zero.

5. Within each iteration of the loop:

- We calculate the least significant bit (LSB) of the binary representation by taking the remainder of the decimal number divided by 2.
- We add this LSB, multiplied by the current position `p`, to the `sum`.
- We update the decimal number by removing its LSB (by integer division by 2), as we've already processed it.
- We update the position `p` by multiplying it by 10, moving to the next position to the left.

6. After the loop completes, `sum` contains the binary representation of the decimal number.

7.This logic iteratively extracts the binary digits of the decimal number, starting from the least significant bit, and constructs the binary representation bit by bit.

Let's trace the program with a numerical example :

Let's convert the decimal number 13 to binary:

  1. Initialize p as 1 and sum as 0.

  2. In the first iteration:
    • deci = 13
    • Calculate (13 % 2) * p = (1) * 1 = 1
    • Add the result to sum: sum = 0 + 1 = 1
    • Update deci by integer division: deci = 13 / 2 = 6
    • Update p: p = 10

  3. In the second iteration:
    • deci = 6
    • Calculate (6 % 2) * p = (0) * 10 = 0
    • Add the result to sum: sum = 1 + 0 = 1
    • Update deci: deci = 6 / 2 = 3
    • Update p: p = 100

  4. In the third iteration:
    • deci = 3
    • Calculate (3 % 2) * p = (1) * 100 = 100
    • Add the result to sum: sum = 1 + 100 = 101
    • Update deci: deci = 3 / 2 = 1
    • Update p: p = 1000

  5. In the fourth iteration:
    • deci = 1
    • Calculate (1 % 2) * p = (1) * 1000 = 1000
    • Add the result to sum: sum = 101 + 1000 = 1101
    • Update deci: deci = 1 / 2 = 0
    • Update p: p = 10000

  6. The loop terminates since deci becomes 0.

  7. The final binary equivalent is 1101.

Time and Space Complexity:

Time Complexity:O(log N), where N is the decimal number.
In each iteration of the loop, the decimal number is divided by 2. The number of iterations required is proportional to the number of bits required to represent the decimal number in binary, which is log(N) base 2.

Space Complexity:O(1) because the algorithm uses a constant amount of extra space, regardless of the input size. This constant space is used to store variables such as p, sum, and deci.
