Duplicate Characters in a String, Bitwise Operators

Problem Statement:

Read a string from the user (all lowercase alphabates) and figure out the duplicate characters or characters occured twice or more times in the string using Bitwise Operators.

Illustration:

Example:

Enter String: kushalvardhan

Output : h is a duplicate
a is a duplicate
a is a duplicate

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Bitwise Operators (dont worry we explained them below)
  • Ascii Table, Character to integer conversion
  • Looping Constructs
  • Conditional Statements (if-else)

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>

int main()
{
    char str[80];
    int i, h=0, x=0;

    printf("Enter your string : ");
    gets(str);

    for(i=0; str[i]!='\0'; ++i)
    {
        x=1;
        x = x<<str[i]-97;

        if(h&x)
            printf("%c is a duplicate\n", str[i]);
        else
            h=x|h;
    }
    return 0;
}

Python Solution :


def main():
    string = input("Enter your string: ")
    h = 0
    for char in string:
        position = ord(char) - ord('a')
        x = 1 << position
        if h & x:
            print(char, "is a duplicate")
        else:
            h = x | h

if __name__ == "__main__":
    main()

Left Shift Operator (<<)

The left shift operator (<<) is a binary operator that shifts the bits of a number to the left by a specified number of positions.

How it Works:

  1. Operand: The left operand is the number whose bits are to be shifted.
  2. Number of Positions: The right operand specifies the number of positions to shift the bits to the left.

For example, consider the binary representation of the number 5:

5 (decimal) = 00000101 (binary)

Now, let's apply the left shift operator (<<) to shift the bits of 5 to the left by 2 positions:

5 << 2

Here's what happens:

  1. Each bit in the binary representation of 5 is shifted to the left by 2 positions.
  2. Zeros are filled in from the right to fill the empty positions created by the shift.

The result of shifting 5 to the left by 2 positions is:

5 (decimal) = 00000101 (binary)
              -------------------
              00010100 (binary)

Now, let's convert the resulting binary number back to decimal:

00010100 (binary) = 20 (decimal)

So, 5 << 2 equals 20.

In summary, the left shift operator (<<) shifts the bits of a number to the left by a specified number of positions, effectively multiplying the number by 2 to the power of the specified number of positions.

Bitwise AND and OR Operators

Bitwise AND (&)

Bitwise AND (&) is a binary operation that takes two equal-length binary representations and performs the logical AND operation on each pair of corresponding bits. The result in each position is 1 if both bits are 1; otherwise, the result is 0.

For example, 5 & 3:

  • Binary representation of 5: 0101
  • Binary representation of 3: 0011
  • Performing bitwise AND:
  • 0101 (5) & 0011 (3) = 0001 (1)

Bitwise OR (|)

Bitwise OR (|) is a binary operation that takes two equal-length binary representations and performs the logical OR operation on each pair of corresponding bits. The result in each position is 1 if at least one of the bits is 1; otherwise, the result is 0.

For example, 5 | 3:

  • Binary representation of 5: 0101
  • Binary representation of 3: 0011
  • Performing bitwise OR:
  • 0101 (5) | 0011 (3) = 0111 (7)

Explanation:

Explanation of Finding Duplicates using Bitwise Operators:

  1. Initialize the Hash Variable (`h`): We start by initializing a variable `h` to 0. This variable will serve as a "hash" to keep track of the characters we have encountered.

  2. Iterate Through the String: We loop through each character in the input string.

  3. Calculate Bit Position: For each character, we need to determine its position in the alphabet. We do this by subtracting the ASCII value of 'a' from the ASCII value of the current character. This gives us a number between 0 and 25, corresponding to the character's position in the alphabet.

  4. Create a Mask with Left Shift Operator (`<<`): We use the left shift operator (`<<`) to create a bitmask for the current character. This operator shifts the binary representation of the number `1` to the left by the number of positions given by the character's position in the alphabet. For example, if the character is 'a' (position 0), `1 << 0` gives us `00000001`, if it's 'b' (position 1), `1 << 1` gives us `00000010`, and so on.

  5. Check for Duplicates: We use the bitwise AND operator (`&`) to check if the bit at the calculated position is already set in the hash variable `h`. If the result of the bitwise AND operation is non-zero, it means that the bit is already set, indicating that we have encountered the character before and it's a duplicate.

  6. Update the Hash Variable (`h`): If the character is not a duplicate, we use the bitwise OR operator (`|`) to set the bit at the calculated position in the hash variable `h`. This marks the character as encountered, allowing us to detect duplicates later in the string.

  7. Repeat for Each Character: We repeat steps 3-6 for each character in the string.

Solved Example

Let's solve for the input string "hello".

  • For the first character 'h':
    • The position of 'h' in the alphabet is 7.
    • Bitwise left shift sets the 8th bit to 1: x = 1 << 7 = 128.
    • Since 'h' is the first occurrence, 'h' is updated to 128.


  • For the second character 'e':
    • The position of 'e' in the alphabet is 4.
    • Bitwise left shift sets the 5th bit to 1: x = 1 << 4 = 16.
    • Since 'e' is the first occurrence, 'e' is updated to 16.


  • For the third character 'l':
    • The position of 'l' in the alphabet is 11.
    • Bitwise left shift sets the 12th bit to 1: x = 1 << 11 = 2048.
    • Since 'l' is the first occurrence, 'l' is updated to 2048.


  • For the fourth character 'l':
    • The position of 'l' in the alphabet is 11.
    • Bitwise left shift sets the 12th bit to 1: x = 1 << 11 = 2048.
    • Since 'l' was already encountered, it is considered a duplicate.
    • The program prints: 'l is a duplicate'.


  • For the fifth character 'o':
    • The position of 'o' in the alphabet is 14.
    • Bitwise left shift sets the 15th bit to 1: x = 1 << 14 = 16384.
    • Since 'o' is the first occurrence, 'o' is updated to 16384.

Time and Space Complexity:

The time complexity of the given logic is O(n), This complexity arises because the program iterates through each character of the input string once.

The space complexity of the given logic is O(n), where n is the size of the string.
