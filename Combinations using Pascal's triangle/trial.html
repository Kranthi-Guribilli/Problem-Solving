nCr Using Pascal's triangle

Problem Statement:

Implement a Recursive program to calculate the combination nCr using Pascal's triangle.

Illustration:

Input:

Enter n and r to find nCr : 5 2

Explanation: We want to find 5 choose 2 (5C2) using Pascal's triangle. Pascal's triangle: 

                          1
                        1   1
                      1   2   1
                    1   3   3   1
                  1   4   6   4   1
                1   5  10  10   5   1
Here's how we use Pascal's triangle to calculate nCr:

1) Each row in Pascal's triangle corresponds to a value of n.
2) Each element in a row corresponds to a value of r.
3) The value at row n and position r (0-indexed) represents the value of nCr.
4) To find the value of nCr, we start at the top of the triangle and count down n rows and over r positions.
5) The number at that position is the value of nCr.

For example, to find 5C2:
we start from row 5 and count down 2 positions. (0-indexed)
The value at row 5, position 2 is 10, which is the result.

Output: 5C2 is : 10

Prerequisites:

To understand and implement the solution, you should be familiar with the following concepts:

  • Familiarity with recursion and recursive functions.
  • Knowledge of how Pascal's triangle is constructed and its properties.
  • loops and conditional statements.

Solution:

Classic C Program:


#include <stdio.h>
#include <stdlib.h>

int ncr(int n, int r);

int main()
{
    int n, r;

    printf("Enter n and c to find nCr : ");
    scanf("%d %d", &n, &r); //r must be less than or equal to n

    printf("\n %dC%d is : %d\n", n, r, ncr(n, r));

    return 0;
}

int ncr(int n, int r)
{
    if(r==0 || n==r)
        return 1;
    else
        return ncr(n-1, r-1) + ncr(n-1, r);
}

Python Solution :


def ncr(n, r):
    if r == 0 or n == r:
        return 1
    else:
        return ncr(n - 1, r - 1) + ncr(n - 1, r)

def main():
    n = int(input("Enter n: "))
    r = int(input("Enter r: "))

    print(f"{n}C{r} is: {ncr(n, r)}")

if __name__ == "__main__":
    main()

Explanation:

1. The program prompts the user to input two integers, n and r, representing the values for calculating nCr.
2. It then calls the ncr function with these inputs.
3. The ncr function recursively calculates the binomial coefficient using the formula nCr = (n-1)C(r-1) + (n-1)Cr.
4. Base cases are checked: if r is 0 or n is equal to r, then nCr is 1.
5. Otherwise, the function recursively calculates nCr by summing the results of (n-1)C(r-1) and (n-1)Cr.
6. Finally, the result is returned and printed in the main function.
7. The result represents the number of combinations of choosing r elements from a set of n elements.

How Recursive calls happening here....

Recursion Illustration:

Recursion is a process in which a function calls itself as a subroutine.

In the context of calculating nCr recursively:

  • Each recursive call breaks down the problem into smaller subproblems.
  • These subproblems are solved by calling the same function with modified parameters.
  • The base case, where the recursion stops, is when r equals 0 or n equals r.
  • At this point, the function returns 1, indicating the end of the recursive calls.
  • By solving smaller instances of the problem, the function eventually reaches the base case and computes the final result.

Here's a visualization of the recursive calls for n=5 and r=2:

                     nCr(5,2)
                   /          \
           nCr(4,1)           nCr(4,2)
          /       \           /        \
     nCr(3,0)  nCr(3,1)   nCr(3,1)   nCr(3,2)
     /   \      /     \   /     \      /     \
   ...   ...  ...    ... ...   ...    ...   ...

The function branches out into multiple recursive calls until it reaches the base case.

Time and Space Complexity:

Each recursive call in the nCr function generates two more recursive calls, except for the base cases.
This results in a binary tree-like recursive call structure, with a depth of n.
At each level of recursion, there are two branches (corresponding to the two recursive calls).
Therefore, the total number of recursive function calls is proportional to 2^n. Hence, the time complexity of the nCr function is O(2^n).

As for space complexity, it also depends on the depth of the recursion. O(n)
since there are no additional data structures or arrays being created in each function call, the only additional space used is for maintaining the function call stack.
